
What is server
	= 음식점에서 서빙하는사람
	네이버 서버에게 웹소설 페이지 요청 -> 갖다줌(응답)
	요청을 받으면 요청한 내용을 보내주는 프로그램 (요청을 처리할수 있는 기계)
    
요청 -> http 요청. 서버의 기초
	1. 읽기(GET) : 나 이런 페이지 읽고싶음 이라는 요청 가능
	2. 쓰기(POST) : 댓글, 블로그 포스트 등 작성. 이거 글좀 올려주세요~
	3. 수정(PUT) : 뭔가 수정할 떄
	4. 삭제(DELETE) : 뭔가 삭제할때

JavaScript : html 조작과 변경을 위한 언어
	웹페이지를 다이나믹하게 바꿔주기 가능.
	자바스크립트 해석은 크롬, 파이어폭스 등의 브라우저가 담당
	크롬이 V8해석 엔진을 만들었는데 
	이걸 똑 떼와서 만든게 nodeJS

Node.js
	브라우저 내에서 말고도 다른 환경에서도 javaScript 실행할 수 있게 도와줌
	자바스크립트 실행창, 실행 환경임
	덕분에 자바 스크립트를 프로그래밍 언어처럼 사용할 수 있음
	왜 nodeJS로 서버를 만들까 -> 	Non-blocking I/O가 좋음
	코드가 매우 짧고 쉬워서 빠른 개발 가능
	but, 만들게 웹서비스가 아닌 경우는 좋지 않을 수 있음..!

Non-blocking I/O

	일반 언어로 만든 서버 : 중간에 버거운 요청을 만나면 다른 요청을 못받고 중지됨
	nodeJS로 만든 서버 : 요청을 다 받고 처리속도가 빠른 요청부터 처리함
		-> 이게 Non-blocking I/O. 
	SNS, 채팅 서비스와 같이 요청이 많은 경우에 좋음
	일반 서버로도 가능. 스케일링 혹은 멀티쓰레딩을 통해서..!
